# 【定義書】数式処理システム SymPy の AST 構造に基づく微分公式の分類基準

## 目次

本定義書は、数式処理ライブラリ **SymPy** が生成する **抽象構文木 (Abstract Syntax Tree, AST)** のノード構造に基づき、微分公式の適用条件を厳密に規定するものである。

---

## 凡例: AST 構造の表記法

- `［Class］` : ノードの型（Main Operator）
- `-> (arg1, arg2, …)` : そのノードが持つ引数（子ノード）
- `{condition}` : 判定の決め手となる属性

---

## 1. 対数微分法 (Logarithmic Differentiation)

### 定義

底と指数の両方に変数 $x$ を含むべき乗形式。

### AST 構造による説明

- ルートが `［Pow］`（べき乗）であり，
- その第 1 引数（底）と第 2 引数（指数）の **両方** が，下層に `Symbol('x')` を持っている構造。

### SymPy 基準

```python
isinstance(expr, Pow) and base.has(x) and exp.has(x)
```

### 具体例（TRUE）

- $x^x$ → `［Pow］ -> (x, x)`
- $x^{\sin x}$ → `［Pow］ -> (x, ［sin］->(x))`

### 除外例（FALSE）

- $e^x$ → `［exp］ -> (x)`
    - 理由: AST ルートが `Pow` ではなく `exp` 関数クラスであるため
- $x^2$ → `［Pow］ -> (x, 2)`
    - 理由: 第 2 引数が `Integer` であり $x$ を含まないため

### 教育的曖昧さ (Ambiguity)

- $x^{ln x}$: 対数微分法が適しているが，$ln x = t$ と置換する手法もある。
    
    本定義では **AST 構造を優先し「対数微分法」** とする。

## 2. 積の微分 (Product Rule)

### 定義

変数 $x$ を含む 2 つ以上の関数の積。

### AST 構造による説明

- ルートが `［Mul］`（乗算）であり，
- その子ノードリストの中に「定数 (`Number`) ではなく，かつ $x$ を含むノード」が **2 個以上** 存在する構造。

### SymPy 基準

```python
isinstance(expr, Mul) and count(args having x) >= 2
```

### 具体例（TRUE）

- $x \sin x$ → `［Mul］ -> (x, ［sin］->(x))`
    - $x$ を含む項が 2 つ → 積の微分を適用
- $x^2 (x+1)$ → `［Mul］ -> (［Pow］->(x, 2), ［Add］->(x, 1))`

### 除外例（FALSE）

- $5x^2$ → `［Mul］ -> (5, ［Pow］->(x, 2))`
    - 理由: $x$ を含む項が `Pow` の 1 つだけで，もう 1 つは `Integer(5)` であるため

### 教育的曖昧さ (Ambiguity)

- $dfrac{x}{2}$: `［Mul］ -> (x, 1/2)` となるが，定数倍として扱うため **除外**。
- $dfrac{x}{x+1}$: `［Mul］ -> (x, (x+1)^{-1})` となり，構造上は「積」の条件を満たす。
    
    本研究では **「商」を優先** するが，広義の「積」としても検出される仕様とする。

## 3. 商の微分 (Quotient Rule)

### 定義

分母に変数 $x$ を含む分数形式。

### AST 構造による説明

- SymPy には `Div` ノードがないため，「逆数の積」として表現される。
- ルートが `［Mul］` であり，子ノードに **「指数が負の整数である `［Pow］`」** が含まれている構造。
    
    または，ルート自体がそのような `［Pow］` である構造。
    

### SymPy 基準

```python
(expr is Mul and has Pow(..., -n)) or (expr is Pow(..., -n))
```

### 具体例（TRUE）

- $\dfrac{x}{x+1}$ →
    
    `［Mul］ -> (x, ［Pow］->(［Add］->(x, 1), -1))`
    
    - 第 2 引数が $-1$ 乗なので商とみなす
- $\dfrac{1}{x^2}$ →
    
    `［Pow］ -> (x, -2)`
    

### 除外例（FALSE）

- $\dfrac{x}{2}$ → `［Mul］ -> (x, 0.5)`
    - 理由: 分母が定数であり，負の指数の `Pow` が存在しないため

### 教育的曖昧さ (Ambiguity)

- $x^{-1}$: 単項なら $x^n$ の公式で処理可能だが，構造上は「商（逆数）」である。

## 4. 三角関数の微分 (Trigonometric Functions)

### 定義

基本的な三角関数。

### AST 構造による説明

- ルートノードのクラス型（`func`）が，特定の三角関数クラスである構造。

### SymPy 基準

```python
expr.func in [sin, cos, tan]
```

### 具体例（TRUE）

- $\sin x$ → `［sin］ -> (x)`
- $\cos(x^2)$ → `［cos］ -> (［Pow］->(x, 2))`

### 除外例（FALSE）

- $\sin^{-1} x$ → `［asin］ -> (x)`
    - 理由: クラスが `asin` であり対象リスト外

---

## 5. ネイピア数の指数関数 (Exponential – Base $e$)

### 定義

底が $e$ の指数関数。

### AST 構造による説明

- ルートノードのクラス型が `［exp］` である構造。
    
    （SymPy では $e^x$ は `Pow` ではなく，専用の `exp` 関数として扱われる。）
    

### SymPy 基準

```python
expr.func is exp
```

### 具体例（TRUE）

- $e^x$ → `［exp］ -> (x)`
- $e^{x^2}$ → `［exp］ -> (［Pow］->(x, 2))`

### 除外例（FALSE）

- $2^x$ → `［Pow］ -> (2, x)`
    - 理由: クラスが `exp` ではなく `Pow` であるため

---

## 6. 一般の指数関数 (General Exponential)

### 定義

底が $e$ 以外の定数の指数関数。

### AST 構造による説明

- ルートが `［Pow］` であり，
- 第 1 引数（底）が定数，
- 第 2 引数（指数）が変数 $x$ を含む構造。

### SymPy 基準

```python
isinstance(expr, Pow) and [base.is](http://base.is)_Number and exp.has(x)
```

### 具体例（TRUE）

- $2^x$ → `［Pow］ -> (2, x)`

### 除外例（FALSE）

- $e^x$ → `［exp］ -> (x)`
    - 理由: `Pow` クラスではないため
- $x^2$ → `［Pow］ -> (x, 2)`
    - 理由: 底が変数で指数が定数のため

---

## 7. 対数関数の微分 (Logarithmic Functions)

### 定義

自然対数および一般の対数。

### AST 構造による説明

- ルートノードのクラス型が `［log］` である構造。

### SymPy 基準

```python
expr.func is log
```

### 具体例（TRUE）

- $\ln x$ → `［log］ -> (x)`
- $\log_{10} x$ → `［log］ -> (x, 10)`

---

## 8. べき乗微分 (Power Rule)

### 定義

$x^n$（$n$ は整数）の形式。

### AST 構造による説明

- ルートが `［Pow］` であり，
- 第 1 引数が純粋なシンボル `x`，
- 第 2 引数が「整数 (`Integer`)」である構造。

### SymPy 基準

```python
isinstance(expr, Pow) and base == Symbol('x') and [exp.is](http://exp.is)_Integer
```

### 具体例（TRUE）

- $x^2$ → `［Pow］ -> (x, 2)`
- $x^{-3}$ → `［Pow］ -> (x, -3)`

### 除外例（FALSE）

- $(2x)^2$ → `［Pow］ -> (［Mul］->(2, x), 2)`
    - 理由: 底が `x` ではなく `2x` (`Mul`) であるため → 合成関数へ
- $x^{1/2}$ → `［Pow］ -> (x, 0.5)`
    - 理由: 指数が整数ではなく有理数のため → 累乗根へ

---

## 9. 累乗根の微分 (Derivative of Roots)

### 定義

$\sqrt{x}$ や $x^{\frac{1}{3}}$ など。

### AST 構造による説明

- ルートが `［Pow］` であり，
- 第 2 引数（指数）が「整数ではない有理数（分数や小数）」である構造。

### SymPy 基準

```python
isinstance(expr, Pow) and not [exp.is](http://exp.is)_Integer
```

### 具体例（TRUE）

- $\sqrt{x}$ → `［Pow］ -> (x, 1/2)`
    - SymPy はルートを $\frac{1}{2}$ 乗として保持する
- $\sqrt[3]{x^2 + 1}$ → `［Pow］ -> (x^2 + 1, 1/3)`

### 除外例（FALSE）

- $x^2$ → `［Pow］ -> (x, 2)`
    - 理由: 指数が整数のため

---

## 10. 合成関数の微分 (Chain Rule)

### 定義

関数の中に別の関数が入っている構造。

### AST 構造による説明

- 関数ノード（`sin`, `exp` など）や演算ノード（`Pow`）の「子ノード（引数）」が，
- 単純な葉ノード `x` ではなく，さらに枝分かれした部分木（$x$ を含む式）を持っている構造。

### SymPy 基準

```python
any(arg.has(x) and not isinstance(arg, Symbol) for arg in expr.args)
```

### 具体例（TRUE）

- $\sin(x^2)$ → `［sin］ -> (［Pow］->(x, 2))`
    - 引数が `x` ではなく `Pow` ノードである
- $e^{2x}$ → `［exp］ -> (［Mul］->(2, x))`
- $(2x+1)^3$ → `［Pow］ -> (［Add］->(2x, 1), 3)`

### 除外例（FALSE）

- $\sin x$ → `［sin］ -> (x)`
    - 理由: 引数が単純なシンボル `x` であるため
- $x^2$ → `［Pow］ -> (x, 2)`
    - 理由: `Pow` の底が単純なシンボル `x` であるため特例除外

### 教育的曖昧さ (Ambiguity)

- $(2x)^2$:
    
    数学的には $4x^2$ だが，AST 上は `Pow(Mul(2, x), 2)` なので，**合成関数** として検出される。

## 定義書の修正と今後の課題

本研究における「微分公式の定義」が、どのように確立されてきたかの経緯をまとめる。

1. 初期段階：ヒューリスティックな定義
    - 当初は「見た目が掛け算なら積の微分」といった、直感的な文字列パターンに近い基準で分類を行っていた。
    - 課題: 5x^2（定数倍）を積と判定したり、√x の扱いが曖昧であるなど、一貫性に欠けていた。
2. 中期段階：SymPy仕様への準拠
    - Pythonライブラリ SymPy の内部クラス（Mul, Pow, Add）を基準に採用。
    - 発見: x/y が Mul(x, Pow(y, -1)) と表現されるなど、計算機の論理と教育的直感のズレ（Ambiguity）が判明。
    - 対応: 「商の微分」を「負の指数を持つ Pow を含む Mul」と再定義し、計算機仕様を受け入れる方針へ転換。
3. 最終段階：AST構造に基づく厳密定義（現在）
    - 学術的根拠（SICP, Structure Sense）を取り入れ、定義を「ASTのノード型」で記述。
    - 修正点:
        - 「ルート」→「累乗根」への名称変更（数学的正確性のため）。
        - 「一般指数関数 (a^x)」を「ネイピア数 (e^x)」から分離独立（公式の違いを反映）。
        - 各定義に「TRUE条件」だけでなく「FALSE条件（境界事例）」と「AST構造の説明」を付記し、判定ロジックの透明性を確保。
